Pygame Puzzle-Adventure Game

**Python ‚Ä¢ Pygame ‚Ä¢ Object-Oriented Design (OOP) ‚Ä¢ Algorithms ‚Ä¢ Game Engine**

This project is a **top-down, 2D tile-based puzzle-adventure game** developed in **Python** using the **Pygame** library. Its primary goal was to create a complete, multi-level game experience from the ground up, demonstrating a **scalable engine** and a wide variety of integrated gameplay mechanics.

The "problem" solved by this architecture is the challenge of seamlessly integrating diverse puzzle types (logic, timing, memory, physics) and action elements (enemy AI, hazards) into a single, cohesive game framework.

‚öôÔ∏è Core Technical Architecture

Feature Technical Implementation
**Scalable Level Engine** The main.py file acts as the game's shell, using a Level Manager to dynamically load self-contained level classes (e.g., Level1, Level2) from the levels/ directory. This allows for **easy, modular expansion** and level creation.
**Centralized Game Objects** The game_objects.py file defines all interactive elements as classes, ensuring they are **reusable, self-contained units** that enforce the single responsibility principle.
**Dynamic Player Controller** The Player class handles all movement and collision and supports two distinct movement paradigms:
  • **Hold-to-Move:** Smooth, continuous movement for action/timing levels.
  • **Press-per-Move:** A turn-based, grid-locked system essential for precise, Sokoban-style **Boulder** puzzles (used in Levels 5 and 8).
**AI and Pathfinding** Implementation of intelligent hazards like PatrolEnemy and ChaserEnemy, with the latter using a simple pathfinding **AI** to dynamically track the player across the grid.

‚ú® Key Game Mechanics Demonstrated

The system incorporates a wide range of mechanics‚Äîfrom static puzzle elements to dynamic hazards‚Äîall managed by their respective classes:

  • **Hazards:** PatrolEnemy and ChaserEnemy (AI), Bridge (disappears on a timer), and Gear (spinning collision hazard).
  • **Logic Puzzles:** Key, Switch (timed and sequence-based), Mirror (rotatable), and the LightBeam (a dedicated class for real-time collision and reflection logic).
  • **Physics/Manipulation:** Boulder (pushable) and PressurePlate (triggered by the boulder's weight).

üó∫Ô∏è Multi-Level Learning Structure (8 MVP Levels)

Each of the eight MVP levels acts as a self-contained module to introduce and test a new mechanic, culminating in a comprehensive "final exam" for the player.

Level Primary Mechanic Focus
**Level 1** Maze Navigation & Key/Door (Basic interaction).
**Level 2** Timed Switch & Gate (Precision and timing).
**Level 3** Light Beam & Mirror Puzzle (Spatial logic and reflection).
**Level 4** Memory/Sequence Puzzle (Logic flow).
**Level 5** Boulder & Pressure Plate Puzzle (Push-box style physics).
**Level 6** Timed Bridge Navigation (Dynamic terrain and timing).
**Level 7** Hazard Avoidance (Rotating Gears and enemy paths).
**Level 8** **Synthesis/Final Gauntlet** (Integration of all previous mechanics and movement modes).
