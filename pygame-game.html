Pygame Puzzle-Adventure Game

**Python ‚Ä¢ Pygame ‚Ä¢ Object-Oriented Programming (OOP) ‚Ä¢ Algorithms**

This project is a **2D grid-based puzzle game** developed in **Python** using the **Pygame** library. It was specifically built to demonstrate strong command of **Object-Oriented Programming (OOP)**, modular level design, and the implementation of complex, interconnected game mechanics and algorithms.

The player progresses through a series of levels that sequentially introduce new objects and rules, culminating in a final stage that integrates all learned mechanisms.

Feature Technical Implementation
**Modular Architecture** A robust OOP structure utilizing dedicated classes (Level, Player, Mirror, Gear, etc.). This design enforces **separation of concerns**, making the codebase maintainable and easily extensible.
**Reflection Logic** Implementation of a dynamic **Light Beam** system (LightBeam) with recursive path calculation. The system determines the beam's next path based on the mirror's precise **angle** (45¬∞ or 135¬∞), simulating real-time light reflection.
**Advanced AI and Pathfinding** The **Chaser Enemy** (ChaserEnemy) utilizes the **A*-Search Algorithm** to dynamically calculate the optimal path to the player, while efficiently navigating around all map obstacles.
**Complex Interactions** Managed inter-object interaction, such as the physics-based movement of **Boulders** and their interaction with **Pressure Plates**, which activate conditional level mechanics.
**Dynamic Hazards** Precise management of game timing and collision logic for hazards, including rotating **Gears** and alternating **Bridges**, requiring the use of cooldowns and timer systems.

üß© Challenges & Solutions

Real-Time Light Beam Reflection

The most significant technical challenge was designing the **Light Beam Reflection Logic**. Because the mirrors can be rotated by the player in real-time, the light beam's path must be calculated on every frame, chaining reflections across multiple mirrors until it hits a wall or the target receiver.

  • **Solution:** A **recursive function** was implemented within the LightBeam class. This function efficiently handles the reflection loop: it takes the beam's current position and direction, checks for a collision with a mirror, calculates the new angle based on the mirror's orientation (45¬∞ or 135¬∞), and calls itself to continue the beam until termination.

Object and Collision Management

Managing the collision bounds and interaction logic for the many different types of objects (player, rocks, enemies, gears, light beams) required a highly organized approach to avoid complex, monolithic code.

  • **Solution:** Strict adherence to OOP principles. All interactive objects inherit from a common **Base Object Class**, overriding methods like update() and handle_collision() to manage their unique behaviors, ensuring the main game loop remains clean and maintainable.
